01 ทำไม Options Trading ต้องการ Pricing Model?

Natenberg เริ่มด้วยข้อสังเกตสำคัญ: นักเทรด futures สนใจแค่ทิศทาง — ถ้าถูก ก็กำไร แต่ นักเทรด options ต้องถูกทั้งทิศทาง และ speed ของตลาด

Futures Trader

ถ้าเชื่อว่าราคาขึ้น → ซื้อ → ถูก = กำไร ง่ายๆ ไม่สนใจ speed

Options Trader

ถ้าเชื่อว่าราคาขึ้น → ซื้อ call → ต้องถูกทั้ง ทิศทาง และ ราคาขึ้นเร็วพอก่อน expiry

⚑ สาเหตุหลักที่ speculator ขาดทุนใน options

ซื้อ call เพราะมั่นใจว่าหุ้นขึ้น — หุ้นขึ้นจริง แต่ขึ้นช้าเกินไป time value decay เร็วกว่า intrinsic value ที่เพิ่มขึ้น ผลลัพธ์คือ option มีมูลค่าน้อยลงทั้งที่หุ้นขึ้น

💡 Key Insight จาก Natenberg

ถ้าคุณเก่งแค่ predict ทิศทาง — เทรด futures หรือหุ้นตรงๆ ดีกว่า options จะเหมาะเมื่อคุณมีความสามารถ predict speed / volatility ของตลาดด้วย

02 Expected Return → Theoretical Value · หัวใจของ Pricing

Natenberg ใช้ตัวอย่างลูกเต๋า 6 หน้า เพื่ออธิบาย concept ที่เป็นรากฐาน:

Expected Return = Σ (Probability × Outcome)
ค่าเฉลี่ยถ่วงน้ำหนักของทุก outcome ที่เป็นไปได้
ตัวอย่างลูกเต๋า

ลูกเต๋า 6 หน้า หน้าละ 1/6 probability → Expected Return = (1+2+3+4+5+6)/6 = $3.50
ถ้าจ่าย <$3.50 เพื่อโยก → คาดกำไรในระยะยาว · ถ้าจ่าย >$3.50 → คาดขาดทุน

แต่ในโลกจริงมี Carrying Cost (ดอกเบี้ย) ด้วย ดังนั้น:

Theoretical Value = Expected Return − Carrying Cost
ราคาที่ยุติธรรมที่ทำให้ทั้ง buyer และ seller break even ในระยะยาว

"The goal of option evaluation is to determine, through theoretical pricing models, the theoretical value of an option. The trader can then make an intelligent decision whether the option is overpriced or underpriced in the marketplace."

— Natenberg, Chapter 3
💡 Edge = ซื้อ underpriced / ขาย overpriced

ถ้า Theoretical Value = $6 แต่ตลาดขาย option ที่ $4 → ซื้อ มี edge $2 · ถ้าตลาดขายที่ $8 → ขาย มี edge $2 — นี่คือ logic พื้นฐานที่ professional traders ใช้

03 A Word on Models · โมเดลไม่ใช่ความจริง

"It is unwise, and sometimes dangerous, to assume that the model and the real world which it represents are identical in every way."

— Natenberg, Chapter 3
โมเดลมีค่า

เหมือนเทียนเล่มเล็กในห้องมืด — ไม่ได้เห็นทุกอย่าง แต่ดีกว่าเดินในความมืดสนิท

โมเดลมีขีดจำกัด

Garbage in → Garbage out ถ้า input ผิด โดยเฉพาะ Volatility output จะผิดทั้งหมด

⚑ อันตรายของ over-confidence ใน model

ยิ่ง trade ใหญ่ขึ้น ความผิดพลาดของ model จะถูก magnify ขึ้นตาม — Natenberg เตือนว่า trader ที่ trade ขนาดใหญ่โดยเชื่อ model 100% เสี่ยง "financial disaster" จากความเข้าใจผิดเรื่อง limitation

04 5 ขั้นตอนสร้าง Pricing Model

Natenberg สรุป logic ที่ Black-Scholes ใช้ไว้ใน 5 ขั้นตอน:

1
Propose possible prices at expiration
กำหนดชุดของราคาที่ underlying อาจเป็นได้ที่ expiry — จาก 5 ราคาแบบง่าย จนถึง distribution ต่อเนื่องที่ครอบคลุมทุกความเป็นไปได้
2
Assign probability to each price
ราคาใกล้ราคาปัจจุบันมี probability สูงกว่า → ใช้ lognormal distribution (Black-Scholes assumption) ซึ่งสะท้อนความจริงที่ว่าราคาหุ้นไม่ติดลบ
3
Assume arbitrage-free underlying market
ตลาด underlying ต้องไม่มีโอกาส arbitrage — ทุกคนได้ return ยุติธรรม ไม่มีใครได้เปรียบฟรีๆ จากตลาด underlying
4
Calculate expected return for the option
คำนวณ weighted average ของ option value ที่ทุก price outcome — ผลคือ Expected Return ก่อนหักค่าใช้จ่าย
5
Deduct carrying cost → get Theoretical Value
หักดอกเบี้ยในช่วงที่ถือ option → ได้ Theoretical Value ที่ใช้เปรียบกับราคาตลาดได้จริง

06 Deep Dive · แต่ละ Input มีรายละเอียดอะไรบ้าง
Strike Price

Fixed ตลอดอายุ option — ไม่เปลี่ยน ไม่ต้อง estimate ใส่ตามสัญญาได้เลย

Time to Expiry

ใส่เป็น annualized fraction: 91 วัน = 91/365 ≈ 0.25 · model รับทั้ง calendar days และ trading days

Underlying Price

ใช้ bid price ถ้า long option (จะต้อง hedge ด้วยการ short underlying) · ใช้ ask price ถ้า short option

Interest Rate

ใช้ risk-free rate = T-bill yield ที่ maturity ใกล้เคียงกับ option expiry · สำหรับ stock options มีบทบาท 2 ด้าน (forward price + carrying cost)

Interest Rate มีบทบาท 2 อย่างในหุ้น

1. เพิ่ม forward price → call ได้ประโยชน์, put เสียเปรียบ
2. เพิ่ม carrying cost ของการถือ option → ลดมูลค่า option ทั้งคู่
ผลรวม: rate สูงขึ้น → call value ↑ เล็กน้อย · put value ↓ เล็กน้อย · แต่ effect เล็กมากเมื่อเทียบกับ volatility

07 ⭐ Volatility — Input ที่สำคัญและยากที่สุด

"Of all the inputs required for option evaluation, volatility is the most difficult for traders to understand. At the same time, volatility often plays the most important role in actual trading situations."

— Natenberg, Chapter 3
Inputs อื่น 4 ตัว

Observable ได้จาก market data โดยตรง — ดูจาก screen ได้เลย ไม่ต้อง estimate

Volatility

ต้อง estimate เสมอ — ไม่มีวิธีดูจาก screen โดยตรง ทุกคนเห็น price เหมือนกัน แต่ vol estimate ต่างกัน → สร้าง edge

💡 ทำไม Volatility ถึงสำคัญที่สุด

Volatility เพิ่มขึ้น → ทั้ง Call และ Put มีมูลค่าเพิ่มขึ้นพร้อมกัน — เพราะ vol สูง = โอกาสที่ราคาจะเคลื่อนไหวมาก = โอกาส ITM สูงขึ้นทุก option · นี่คือเหตุผลที่ IV Crush ทำให้ทุก option ราคาตกหลัง earnings

Volatility ในบทนี้ vs บทที่ 4

บทนี้ Natenberg แค่บอกว่า "volatility = speed ของตลาด" และสัญญาว่าจะอธิบาย deep dive ในบท 4 ซึ่งจะครอบคลุม Historical Vol, Implied Vol, และวิธีใช้งานจริง — นั่นคือหัวใจหลักของ options trading

08 Riskless Hedge · แนวคิดที่ทำให้ Black-Scholes ทำงานได้

Black-Scholes สร้างบน assumption ว่าสามารถ hedge option ด้วย underlying ได้แบบ "riskless" — ไม่มีความเสี่ยงสำหรับ small price change:

Hedge Ratio (Delta) = ΔOption / ΔUnderlying
สัดส่วน underlying ที่ต้องถือเพื่อ hedge option position ให้ neutral กับ price change เล็กๆ
ความหมายในทางปฏิบัติ

ถ้า Delta = 0.50 → ถือ call 1 contract ต้อง short หุ้น 50 หุ้น เพื่อ hedge · แต่ Delta เปลี่ยนตลอดเวลาตามราคา → ต้อง rebalance ต่อเนื่อง (dynamic hedging) — นี่คือ concept ที่นำไปสู่ Greeks ทั้งหมดในบท 6
